# LLM Reasoners 代码库快速入门指南

## 🎯 这个代码库是干什么的？

**LLM Reasoners** 是一个用于增强大语言模型（LLM）复杂推理能力的库。

### 核心思想
把**推理问题转化为搜索问题**，用搜索算法找到更好的答案。

```
传统方式：LLM 直接生成答案 → 可能出错 ❌
这个库：LLM + 搜索算法 → 生成多个候选 → 评估每个候选 → 选择最好的 ✅
```

### 解决的问题
- 数学题容易算错
- 逻辑推理容易出错
- 规划问题找不到解
- 多步推理容易迷失方向

## 📚 学习路径（推荐顺序）

### 第一步：理解核心概念（30分钟）

**必读文件：`reasoners/base.py`**

这是整个代码库的**基础抽象层**，定义了三个核心接口：

1. **WorldModel（世界模型）** - 定义状态和状态转换
2. **SearchConfig（搜索配置）** - 定义动作和奖励函数
3. **SearchAlgorithm（搜索算法）** - 实现搜索策略

**关键理解点：**
- 状态（State）：推理过程中的中间表示
- 动作（Action）：从一个状态到另一个状态的操作
- 奖励（Reward）：评估一个动作好坏的值

### 第二步：看一个简单示例（1小时）

**推荐示例：`examples/CoT/blocksworld/`**

这是最简单的完整示例，包含：
- `world_model.py` - 如何定义状态转换
- `search_config.py` - 如何定义奖励函数
- `inference.py` - 如何组合使用

**学习重点：**
1. 如何将问题建模为状态空间
2. 如何定义状态转换规则
3. 如何设计奖励函数

### 第三步：理解搜索算法（45分钟）

**核心文件：`reasoners/algorithm/mcts.py`**

MCTS（蒙特卡洛树搜索）是最强大的搜索算法之一。

**四个阶段：**
1. Selection（选择）- 用 UCT 公式选择路径
2. Expansion（扩展）- 扩展节点，生成动作
3. Simulation（模拟）- 快速模拟到终止
4. Backpropagation（回传）- 将奖励回传到路径

### 第四步：运行交互式示例（30分钟）

**文件：`demo.ipynb`**

实际运行代码，看效果。

## 📖 核心概念详解

### 1. WorldModel（世界模型）

**作用**：定义"状态是什么"和"状态如何转换"

**需要实现的方法：**
- `init_state()` - 初始化状态
- `step(state, action)` - 状态转换
- `is_terminal(state)` - 判断终止条件

**示例（数学题）：**
```python
状态 = "已完成的步骤：2+2=4"
动作 = "然后 4*3=12"
新状态 = "已完成的步骤：2+2=4, 4*3=12"
```

### 2. SearchConfig（搜索配置）

**作用**：定义"有哪些动作"和"如何评估动作"

**需要实现的方法：**
- `get_actions(state)` - 获取可用动作
- `fast_reward(state, action)` - 快速评估（可选但推荐）
- `reward(state, action)` - 完整评估（必需）

**关键设计：**
- `fast_reward`：快速但可能不准确（如用 log-likelihood）
- `reward`：慢但更准确（如验证答案正确性）

### 3. SearchAlgorithm（搜索算法）

**作用**：实现搜索策略

**可选算法：**
- **MCTS**：最强大，适合复杂任务
- **Beam Search**：平衡性能和效果
- **DFS**：深度优先，适合简单任务
- **Greedy**：最快，但效果可能不好

## 💡 使用流程

### 完整流程示例

```python
# 1. 定义 WorldModel
class MyWorldModel(WorldModel):
    def init_state(self): ...
    def step(self, state, action): ...
    def is_terminal(self, state): ...

# 2. 定义 SearchConfig
class MySearchConfig(SearchConfig):
    def get_actions(self, state): ...
    def fast_reward(self, state, action): ...
    def reward(self, state, action): ...

# 3. 选择 SearchAlgorithm
algorithm = MCTS(n_iters=10, depth_limit=5)

# 4. 组合成 Reasoner
reasoner = Reasoner(
    world_model=MyWorldModel(...),
    search_config=MySearchConfig(...),
    search_algo=algorithm
)

# 5. 运行推理
result = reasoner(example)
```

## 🎓 不同应用场景

### 场景 1：数学题（GSM8K）

**状态**：已完成的计算步骤
**动作**：执行下一步计算
**奖励**：计算是否正确

**示例文件**：`examples/CoT/gsm8k/`

### 场景 2：规划问题（Blocksworld）

**状态**：积木的当前排列
**动作**：移动一个积木
**奖励**：是否更接近目标状态

**示例文件**：`examples/CoT/blocksworld/`

### 场景 3：浏览器自动化（BrowserGym）

**状态**：浏览器的当前状态（页面、标签页等）
**动作**：点击、输入、导航等
**奖励**：是否完成任务

**示例文件**：`examples/browsergym/`

## 🔍 常见问题

### Q1: 我应该从哪里开始？

**推荐顺序：**
1. 阅读 `reasoners/base.py` 理解核心抽象
2. 运行 `demo.ipynb` 看实际效果
3. 选择一个示例（如 `examples/CoT/blocksworld/`）深入理解
4. 尝试修改示例，理解每个组件的作用

### Q2: 如何选择搜索算法？

- **简单任务**：Greedy 或 DFS
- **中等任务**：Beam Search
- **复杂任务**：MCTS（最强大但最慢）

### Q3: 如何为自己的任务定制？

1. 定义 State 和 Action 类型
2. 实现 WorldModel（定义状态转换）
3. 实现 SearchConfig（定义动作和奖励）
4. 选择 SearchAlgorithm
5. 组合成 Reasoner 并运行
