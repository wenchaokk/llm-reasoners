## 📚 学习路径

### 第一步：理解核心抽象（最重要！）

#### 1. 核心文件：`reasoners/base.py`

这是整个代码库的**基础抽象层**，定义了三个核心接口：

##### 🔹 WorldModel（世界模型）
```python
class WorldModel(Dynamics, Generic[State, Action, Example]):
    def init_state(self) -> State:        # 初始化状态
    def step(self, state, action) -> State:  # 状态转换
    def is_terminal(self, state) -> bool:    # 判断是否终止
```

**作用**：定义推理问题的"状态空间"和"状态转换规则"

**例子**：
- 数学题：状态 = "当前已完成的步骤"，转换 = "执行下一步计算"
- 积木问题：状态 = "积木的当前排列"，转换 = "移动一个积木"

##### 🔹 SearchConfig（搜索配置）
```python
class SearchConfig(ABC, Generic[State, Action, Example]):
    def get_actions(self, state: State) -> list[Action]:  # 获取可用动作
    def fast_reward(self, state, action) -> tuple[float, dict]:  # 快速奖励（可选）
    def reward(self, state, action, **kwargs) -> tuple[float, dict]:  # 完整奖励
```

**作用**：定义"如何评估一个步骤的好坏"（奖励函数）

**关键点**：
- `fast_reward`：快速评估，用于加速搜索（如用 log-likelihood）
- `reward`：完整评估，可能更慢但更准确（如验证答案正确性）

##### 🔹 SearchAlgorithm（搜索算法）
```python
class SearchAlgorithm(ABC):
    def __call__(self, world_model, search_config, **kwargs) -> AlgorithmOutput:
```

**作用**：实现搜索策略（MCTS、Beam Search、DFS等）

##### 🔹 Reasoner（推理器）
```python
class Reasoner(ABC, Generic[State, Action, Example]):
    def __init__(self, world_model, search_config, search_algo):
        # 组合三个组件
```

**作用**：将三个组件组合在一起，提供统一的推理接口

**1. WorldModel (`world_model.py`)**
```python
class BlocksWorldModel(WorldModel):
    def init_state(self) -> BWState:
        # 从问题中提取初始积木状态
        return BWState(blocks_state=extract_init_state(self.example))
    
    def step(self, state, action) -> tuple[BWState, dict]:
        # 执行动作，更新积木状态
        new_state = update_blocks(state.blocks_state, action)
        # 返回新状态和额外信息（用于奖励计算）
        return new_state, {"goal_reached": check_goal(new_state)}
    
    def is_terminal(self, state) -> bool:
        # 判断是否达到目标状态
        return goal_check(state)
```

**2. SearchConfig (`search_config.py`)**
```python
class BWConfig(SearchConfig):
    def get_actions(self, state) -> list[BWAction]:
        # 根据当前状态，生成所有可能的动作
        return generate_all_actions(state.blocks_state)
    
    def fast_reward(self, state, action) -> tuple[float, dict]:
        # 快速评估：用 LLM 的 log-likelihood 评估动作
        intuition = self.base_model.get_loglikelihood(prompt, [prompt + action])[0]
        return intuition, {'intuition': intuition}
    
    def reward(self, state, action, **kwargs) -> tuple[float, dict]:
        # 完整评估：结合直觉和目标达成度
        goal_reached = kwargs['goal_reached']  # 从 WorldModel.step 传递过来
        return calculate_reward(intuition, goal_reached), {...}
```

**3. 主程序 (`inference.py`)**
```python
# 1. 加载模型
```

## 🎯 关键概念总结

### 1. 状态（State）
- **定义**：推理过程中的中间表示
- **例子**：
  - 数学题：已完成的步骤序列
  - 积木问题：积木的当前排列

### 2. 动作（Action）
- **定义**：从一个状态到另一个状态的操作
- **例子**：
  - 数学题：执行一步计算
  - 积木问题：移动一个积木

### 3. 奖励（Reward）
- **定义**：评估一个动作好坏的值
- **两种类型**：
  - `fast_reward`：快速但可能不准确
  - `reward`：慢但更准确

### 4. 搜索树
- **定义**：搜索过程中探索的状态空间
- **节点**：状态
- **边**：动作
- **根节点**：初始状态
- **叶子节点**：终止状态

## 💡 常见问题

### Q1: WorldModel 和 SearchConfig 的区别？
- **WorldModel**：定义"状态是什么"和"状态如何转换"
- **SearchConfig**：定义"有哪些动作"和"如何评估动作"

### Q2: fast_reward 和 reward 的区别？
- **fast_reward**：快速评估，用于加速搜索（如 log-likelihood）
- **reward**：完整评估，更准确但可能更慢（如验证答案）

### Q3: 如何选择搜索算法？
- **简单任务**：Greedy 或 DFS
- **中等任务**：Beam Search
- **复杂任务**：MCTS（最强大但最慢）

### Q4: 如何为自己的任务定制？
1. 定义 State 和 Action 类型
2. 实现 WorldModel（定义状态转换）
3. 实现 SearchConfig（定义动作和奖励）
4. 选择 SearchAlgorithm
5. 组合成 Reasoner 并运行
