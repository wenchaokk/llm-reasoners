## 🔄 数据流

### 推理流程

```
问题 (Example)
    │
    ▼
┌─────────────────────────────────────────┐
│  Reasoner.__call__(example)             │
└─────────────────────────────────────────┘
    │
    ├─► WorldModel.init_state()  ──► 初始状态
    │
    ├─► SearchConfig.get_actions(state) ──► 可用动作列表
    │
    ├─► SearchConfig.fast_reward(state, action) ──► 快速评估
    │
    ├─► SearchAlgorithm 开始搜索
    │   │
    │   ├─► Selection: 选择路径
    │   ├─► Expansion: 扩展节点
    │   ├─► Simulation: 模拟到终止
    │   └─► Backpropagation: 回传奖励
    │
    ├─► WorldModel.step(state, action) ──► 新状态
    │
    ├─► SearchConfig.reward(state, action) ──► 完整评估
    │
    └─► 返回 AlgorithmOutput（包含最优路径）
```

## 📦 核心组件详解

### 1. WorldModel（世界模型）

```
┌─────────────────────────────────────┐
│         WorldModel                  │
├─────────────────────────────────────┤
│  init_state()                       │
│    └─► 从问题中提取初始状态          │
│                                     │
│  step(state, action)                │
│    ├─► 执行动作                     │
│    ├─► 更新状态                     │
│    └─► 返回 (新状态, 额外信息)      │
│                                     │
│  is_terminal(state)                 │
│    └─► 判断是否达到终止条件         │
└─────────────────────────────────────┘
```

**示例（积木问题）**：
```python
初始状态: "红色积木在桌子上，蓝色积木在红色积木上"
动作: "拿起蓝色积木"
新状态: "红色积木在桌子上，蓝色积木在手中"
```

### 2. SearchConfig（搜索配置）

```
┌─────────────────────────────────────┐
│         SearchConfig                │
├─────────────────────────────────────┤
│  get_actions(state)                 │
│    └─► 根据状态生成所有可能的动作    │
│                                     │
│  fast_reward(state, action)         │
│    ├─► 快速评估（用 log-likelihood）│
│    └─► 返回 (奖励值, 详细信息)     │
│                                     │
│  reward(state, action, **kwargs)    │
│    ├─► 完整评估（验证正确性）       │
│    └─► 返回 (奖励值, 详细信息)     │
└─────────────────────────────────────┘
```

**示例（数学题）**：
```python
状态: "已计算：2+2=4"
动作: "然后 4*3=12"
fast_reward: 0.8 (LLM 认为这个动作很合理)
reward: 1.0 (验证：确实正确)
```

### 3. SearchAlgorithm（搜索算法）

```
┌─────────────────────────────────────┐
│         MCTS (示例)                 │
├─────────────────────────────────────┤
│  __call__(world_model, search_config)│
│    │                                 │
│    ├─► 迭代 n_iters 次              │
│    │   │                             │
│    │   ├─► Selection: 选择路径      │
│    │   │   └─► 用 UCT 公式选择      │
│    │   │                             │
│    │   ├─► Expansion: 扩展节点       │
│    │   │   ├─► 获取所有动作         │
│    │   │   └─► 计算 fast_reward     │
│    │   │                             │
│    │   ├─► Simulation: 模拟         │
│    │   │   └─► 快速模拟到终止       │
│    │   │                             │
│    │   └─► Backpropagation: 回传    │
│    │       └─► 更新 Q 值            │
│    │                                 │
│    └─► 返回最优路径                  │
└─────────────────────────────────────┘
```

## 🌳 搜索树结构

```
                    [初始状态]
                       │
        ┌──────────────┼──────────────┐
        │              │              │
     [动作1]        [动作2]        [动作3]
        │              │              │
    [状态1]        [状态2]        [状态3]
        │              │              │
    ┌───┴───┐      ┌───┴───┐      ┌───┴───┐
    │       │      │       │      │       │
 [动作A] [动作B] [动作C] [动作D] [动作E] [动作F]
    │       │      │       │      │       │
[状态A] [状态B] [状态C] [状态D] [状态E] [状态F]
                                    │
                                [终止状态] ✓
```

**MCTS 会**：
1. 探索多个分支
2. 评估每个节点的 Q 值
3. 选择最优路径（Q 值最高的）

## 🔗 组件交互示例

### 完整推理过程（以积木问题为例）

```
1. 初始化
   WorldModel.init_state()
   └─► 状态: "红色在桌上，蓝色在红色上"

2. 获取动作
   SearchConfig.get_actions(state)
   └─► 动作列表: ["拿起蓝色", "拿起红色", ...]

3. 快速评估
   SearchConfig.fast_reward(state, "拿起蓝色")
   └─► 奖励: 0.7 (LLM 认为合理)

4. 执行动作
   WorldModel.step(state, "拿起蓝色")
   └─► 新状态: "红色在桌上，蓝色在手中"
   └─► 额外信息: {"goal_reached": 0.3}

5. 完整评估
   SearchConfig.reward(state, action, goal_reached=0.3)
   └─► 奖励: 0.5 (结合直觉和目标达成度)

6. 继续搜索...
   (重复步骤 2-5，直到找到最优解)
```

## 🎯 关键设计模式

### 1. 策略模式
- `SearchAlgorithm` 是策略接口
- `MCTS`、`BeamSearch` 等是具体策略
- 可以轻松切换不同的搜索算法

### 2. 模板方法模式
- `WorldModel` 和 `SearchConfig` 定义模板
- 用户实现具体方法
- 框架负责调用流程

### 3. 组合模式
- `Reasoner` 组合三个组件
- 提供统一的接口

## 💡 设计优势

1. **模块化**：每个组件职责清晰
2. **可扩展**：容易添加新的搜索算法或 LLM 后端
3. **可复用**：同一个算法可以用于不同任务
4. **可测试**：每个组件可以独立测试
